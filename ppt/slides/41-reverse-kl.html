06 · REVERSE KL
KEY MECHANISM · WHY REVERSE KL WORKS

The student can choose its own teacher

Math requests align against the math expert. Code requests align against the code expert. This “automatic routing” behavior is the deep reason multi-teacher distillation works here.
Difference in distribution shape
SCENE 01
Math task
The student’s rollout lands near the math mode, and reverse KL pushes it toward the high-probability region of the math expert. Code and agent contributions naturally get ignored.
SCENE 02
Code task
The same student, when prompted with code, rolls out in the code mode and automatically shifts its alignment target. No external controller has to tell it “this is code.”
SCENE 03
Any new task
Add a new expert and capability extends. If you can train a “calligraphy expert,” then adding it to the distillation pool gives the student a new teacher it can align to when that mode appears.
This is the underappreciated property of reverse KL in the multi-teacher setting: you do not need an external router to tell the student which teacher to listen to. The student’s own on-policy distribution becomes the router.
