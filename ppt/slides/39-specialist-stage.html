STAGE 01 · SPECIALIST TRAINING

Train one expert per domain

First use SFT for a base layer, then run GRPO for RL. Each expert is responsible only for its own domain, so the reward signal stays clear and uncompromised. V4 trains more than ten such experts.
Training pipeline for a single expert
01
Lay the base with SFT
Run standard fine-tuning on high-quality supervised data inside that domain to build the basic capability first.
02
Run RL with GRPO
DeepSeek’s Group Relative Policy Optimization removes the need for a separate value model. Relative advantage is computed across sampled groups, which lowers memory use and improves stability. V3, R1, and V4 all build on this lineage.
03
Output: one expert checkpoint
Repeat the full pipeline more than ten times to train math, code, agent, instruction, reasoning, and other experts. Each becomes its own checkpoint.
Innovation · Generative Reward Model
Let the actor become the judge
For tasks that are hard to validate with explicit rules, such as writing quality or helpfulness, V4 flips the usual reward-model setup: it uses a generative model as the judge, and even runs RL on the GRM itself. Evaluation ability and generation ability are optimized together.
