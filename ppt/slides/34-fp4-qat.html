MODULE 04 · FP4 QUANTIZATION-AWARE TRAINING

Push the two heaviest spots all the way down to FP4

QAT is applied in post-training. Both MoE expert weights and the CSA indexer’s query-key path move to MXFP4. The key systems fact is that FP4→FP8 dequantization is lossless, so the existing FP8 pipeline can be reused without rewriting the whole training stack.
Two compression targets
MoE Expert Weights
One of the biggest GPU-memory consumers. Quantizing expert weights cuts memory traffic directly.
CSA Indexer QK
A hotspot in long-context scoring. The top-k selector is a real inference bottleneck.
Dequantization path · why it stays lossless
FP4 (E2M1) → FP8 (E4M3) → FP32 master
FP8 has two more exponent bits than FP4, so its dynamic range is large enough to absorb the FP4 sub-block scale factors completely. Backprop uses STE to pass gradients directly back to the FP32 master weights. RL rollout uses the true FP4-quantized weights, so training-time sampling matches deployment-time behavior.
