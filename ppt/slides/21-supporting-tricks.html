CHAPTER 03 · THE SMALL DETAILS THAT MAKE CSA/HCA ACTUALLY WORK

Beyond the main structure, four small components decide whether it works

Each one looks minor in isolation. Remove any of them and some part of the 1M-context story starts to break. CSA and HCA share this same base layer.
01
SHARED KV · MQA
All heads share one KV set
Each compressed KV entry serves as the key and value for every attention head at once. The more heads you have, the more you save, which pushes KV cache down by another order of magnitude on top of compression.
02
SLIDING WINDOW
Do not compress the most recent n_win tokens
Compression loses local detail, but recent tokens are exactly where local detail matters most. So V4 adds a sliding-window branch that keeps the last n_win tokens uncompressed and feeds them into core attention alongside the compressed blocks.
03
ATTENTION SINK
Give the query an abstain option
Each head gets a learnable sink logit added to the softmax denominator. That lets the total attention weight be less than 1, or even near 0, so the query can abstain instead of forcing itself to attend to noise.
04
PARTIAL ROPE
RoPE only acts on the last 64 dimensions
The remaining dimensions stay position-agnostic and hold more general representations. Combined with the output-side trick of applying RoPE again with position = -i, DeepSeek effectively converts absolute position into a relative-position behavior that remains stable at million-token scale.
