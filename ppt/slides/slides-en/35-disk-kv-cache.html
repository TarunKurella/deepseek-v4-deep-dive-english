MODULE 06 · INFERENCE FRAMEWORK · HETEROGENEOUS KV + DISK SPILL

A bold move: spill KV cache to disk

Hybrid attention makes KV cache heterogeneous: CSA, HCA, SWA, and the indexer all have different sizes, update rules, and eviction patterns. That breaks the assumptions behind PagedAttention. V4 responds with a two-level pool, then pushes shared prefixes all the way to disk.
Analogy
Like a database buffer pool for pages
Hot data stays in memory, warm data sits on SSD, cold data goes to disk. KV cache finally gets treated with the same hierarchy.
Who benefits most
Long-context agents and multi-turn chat
Shared prefixes repeat constantly, so skipping that prefill work directly lowers first-token latency.
Why nobody did this earlier
Without compression, disk could not keep up
CSA/HCA already compress KV to roughly 1/m scale. Only after that reduction does disk I/O become viable.
This is not a paper-style “we propose.” It is the kind of move engineers arrive at only after clearing every other systems constraint first: you get compression first, and only then do you earn the right to talk about disk spill.
