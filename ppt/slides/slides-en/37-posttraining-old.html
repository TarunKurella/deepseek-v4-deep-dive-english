Mixed SFT + RLHF creates three recurring pain points

Every serious post-training team has run into these. Together they explain why V4 ends up rebuilding the RL stage instead of merely tuning it.

PAIN 01
RL is unstable
The student model runs RL directly, so curves shake violently and runs crash often. Every debugging session means rolling back to the last usable checkpoint.

Cost: countless nights staring at a loss curve.
PAIN 02
Tasks fight each other
Math, code, agent, and dialogue all get mixed into one reward. Increase the math weight and code drops. Add more agent data and dialogue gets worse.

Cost: negative transfer becomes almost unavoidable.
PAIN 03
Reward hacking
A static scalar reward model gets gamed quickly. The student learns to optimize for the score surface while real quality quietly degrades.

Cost: you keep patching the student’s shortcuts, and the reward model itself needs constant repair.
