Compress aggressively, but keep dense global visibility

Here m′ is far larger than m, so the sequence gets flattened hard. But there is no sparse filtering step afterward. Every query still scans every compressed block. Fine detail is sacrificed to preserve global coverage.

COMPRESSION
Granularity m′ ≫ m
It uses the same compression mechanism, but the stride is enlarged by roughly an order of magnitude. The paper does not disclose the exact m′, but the reference implementation is around m′ = 1024.
ATTENTION
Dense, not sparse
There is no Lightning Indexer here. Each query runs full MQA over all compressed blocks. Because there are so few blocks left, dense attention is affordable again.
ROLE
Global coarse-grained backup
This path carries broad document-level understanding: cross-section, cross-chapter, long-range dependency structure. It alternates with CSA to form a fine-plus-coarse dual channel.
One clean design intuition sits underneath all this: the coarse path must stay dense. If the coarse representation were sparsified again, global visibility would vanish and HCA would collapse into the same job as CSA.
